# 🧰 toolkit

**Utilities for excellence.** Fernando Kalid's manifold of Claude utilities — **skills**, **agents**, **artifacts**, and **style systems** — versioned in one place and packaged so followers, friends, and the internal team can install them on their own Claude accounts.

> 🤖 Reading this as an LLM or script? Start with [`llms.txt`](llms.txt) and [`catalog.json`](catalog.json) — complete machine-readable indexes of everything here.

## Install

### Claude Code (recommended)

This repo is a native [Claude Code plugin marketplace](https://docs.claude.com/en/docs/claude-code/plugins). Two commands, inside any Claude Code session:

```
/plugin marketplace add Fernando-Kalid/toolkit
/plugin install fk-decks@fk-toolkit
```

Swap `fk-decks` for any bundle in the catalog below (`business-case`, `comms-studio`). Update later with `/plugin marketplace update fk-toolkit`.

### claude.ai / Claude apps

1. Download the zip for the skill you want from the [latest release](../../releases/latest) (or clone and run `bash scripts/package_skills.sh` — zips land in `dist/skills/`).
2. In Claude: **Settings → Capabilities → Skills → Upload skill**, and pick the zip.

### Manual (Claude Code, no marketplace)

Copy any skill folder into `~/.claude/skills/` (available everywhere) or `<your-project>/.claude/skills/` (that project only):

```bash
git clone https://github.com/Fernando-Kalid/toolkit.git && cp -R toolkit/plugins/fk-decks/skills/fk-deck ~/.claude/skills/
```

Full per-surface instructions (teams, API, troubleshooting): [docs/INSTALL.md](docs/INSTALL.md).

## Repository layout

```
.claude-plugin/marketplace.json   Claude Code marketplace manifest (defines the bundles)
plugins/<bundle>/                 One installable plugin per theme
  ├── .claude-plugin/plugin.json  Plugin metadata
  └── skills/<skill>/SKILL.md     One folder per skill — SKILL.md frontmatter is the source of truth
agents/                           Standalone subagent definitions (.md with frontmatter)
artifacts/                        Self-contained HTML artifacts worth keeping
styles/                           Style-sheets, design tokens, brand systems
scripts/                          build_index.py · package_skills.sh · sync_skills.sh
docs/INSTALL.md                   Per-surface install guide
catalog.json · llms.txt           Generated machine/LLM indexes — never hand-edited
CLAUDE.md                         Conventions for humans and agents maintaining this repo
```

## Maintaining

- **Add or edit a skill** → drop/edit the folder under a bundle's `skills/`, then `python3 scripts/build_index.py` to regenerate all indexes.
- **Sync from the Claude account** (after editing skills on claude.ai) → `bash scripts/sync_skills.sh`, review `git diff`, commit.
- **Release** → tag `vX.Y.Z` and push; a GitHub Action attaches per-skill zips to the release for claude.ai users.

Conventions and invariants live in [CLAUDE.md](CLAUDE.md).
