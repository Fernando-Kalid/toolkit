## 1. The strategy kernel (Rumelt)

Every recommendation must contain all three, or it isn't strategy:
- **Diagnosis** — what's really going on; the crux, named. Simplifies a messy reality to its critical few facts.
- **Guiding policy** — the overall approach to the crux. A direction, not a goal.
- **Coherent actions** — steps that reinforce one another and follow from the policy.

**Bad-strategy tests (fail any → rewrite):** Is it fluff (abstract words hiding the absence of content)? Does it dodge the real obstacle? Does it mistake goals ("be #1", "grow 3x") for strategy? Does it list incompatible actions that don't cohere?

## 2. Playing-to-Win cascade (Martin)

Five linked choices; each constrains the next. Answer all five and check they reinforce each other:
1. **What is our winning aspiration?** (purpose framed as winning, not participating)
2. **Where will we play?** (segments, geographies, channels, categories — and where we won't)
3. **How will we win?** (the differentiated value + advantage that beats alternatives *there*)
4. **What capabilities must be in place?** (the few that must be excellent)
5. **What management systems are required?** (what sustains the capabilities)

Phase 2 owns choices 1–3 and names 4. Choice 5 and the build-out belong to Phase 3.

**Model archetypes to weigh:** one-time transactional; subscription/SaaS; usage/consumption; marketplace/take-rate; freemium-to-paid; licensing; services/retainer; productized services; enterprise/seat; ads/data; affiliate; hybrid (e.g., platform + services). Note the classic tradeoffs — recurrence buys valuation and predictability but slows early cash; transactional is fast to cash but re-earns demand each cycle; marketplaces face cold-start but compound via network effects.

## 4. Pricing strategy archetypes and WTP mapping

Decide the **direction and logic** here; leave the number sheet and revenue model to Phase 3.

- **Pricing metric** — what you charge for (seat, usage, outcome, transaction, flat). The best metric aligns price with the value the customer receives and scales as they get more value.
- **Archetype** — value-based (anchor to quantified customer value; usually the highest-ceiling), competitive (anchor to alternatives), cost-plus (floor, rarely the right ceiling), freemium (acquisition engine; needs a real paid trigger), tiered/good-better-best (captures WTP heterogeneity), usage/PAYG (aligns cost to value, less predictable).
- **WTP mapping** — take the willingness-to-pay and price-sensitivity from Phase 1 §3, and position against the range of substitutes from §4. State where you sit (penetration / parity / premium) and *why the customer will pay it* — the value story, not just the number.

## 5. Moat taxonomy (the 7 Powers, condensed)

"How to win" needs a durable advantage. Name the specific power; if you can't, the moat is aspirational:
- **Scale economies** — unit cost falls with size in a way rivals can't match.
- **Network effects** — the product gets more valuable as more use it.
- **Counter-positioning** — a new model incumbents can't copy without damaging their own.
- **Switching costs** — leaving is costly (data, integration, workflow, learning).
- **Branding** — durable trust/identity that commands a premium.
- **Cornered resource** — exclusive access to something valuable (IP, talent, data, supply).
- **Process power** — hard-to-replicate operational excellence built over time.

A "better product" with none of these is a lead, not a moat — say how the lead converts into one of the above before it's competed away.

## 6. Decision scorecard template

Make the judgment inspectable. Weights should sum to 100%. Score 1–5, multiply, sum. Then sanity-check the composite against your gut and reconcile any gap explicitly.

| Criterion | Weight | Score (1–5) | Weighted | Rationale |
|---|---|---|---|---|
| Market attractiveness (size × growth × structure) | 20% | | | |
| Right to win / defensibility | 25% | | | |
| Unit-economics potential | 20% | | | |
| Timing (why now) | 10% | | | |
| Team / resource fit | 15% | | | |
| Risk profile (inverse) | 10% | | | |
| **Composite** | 100% | | **/5** | |

Rough reading (calibrate to context): **≥ 4.0** strong Go signal; **3.0–3.9** conditional Go / scoped bet; **2.0–2.9** Pivot territory; **< 2.0** No-Go. The number informs the call; it does not make it — a single fatal flaw (e.g., no path to positive unit economics) overrides a decent average.

## 7. Conviction and Go / No-Go / Pivot logic

- **Go** — the thesis holds, there's a credible right-to-win, and unit economics can plausibly close. State conviction and the top assumptions Phase 3 must validate.
- **Pivot** — the market is real but the current framing (segment, model, or wedge) is wrong. Specify the *changed* choice, not a vague "adjust."
- **No-Go** — a structural flaw (no moat, hostile industry, broken economics, or demand that isn't there). State it cleanly and give the conditions under which it flips to Go. A crisp No-Go saves more value than a hopeful Go.

**Conviction (High / Medium / Low)** reflects the strength *and verifiability* of the evidence, not enthusiasm. Low conviction on a big prize is a reason to run cheap Phase-3 experiments before committing, not a reason to fake certainty. Always state what would move conviction up a level.
