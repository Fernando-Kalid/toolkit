# Module 3 — Semiotic Architecture

**Role in the system:** designs the sign system through which the meaning claim becomes perceptible, recognizable, and ownable.

## Purpose

Turn intended meaning into working signs: the visual, verbal, sonic, spatial, and behavioral codes that make the entity legible and desirable within its culture. Signs are not decoration — each one is a small machine that must do a job (identify, differentiate, categorize, signal, remind). This module specifies the machines.

## Invoke when

After Module 4 has mapped the belief field (signs draw meaning from the field, not from the designer's taste); when a user asks for a semiotic audit, brand-codes review, or distinctive-asset assessment; when creative work keeps "feeling off" and nobody can say why.

## Required inputs

- Ontology Brief (Module 1) — especially *sells vs. represents* and category logic.
- Cultural Field Map (Module 4) — the codes, myths, and status games already circulating.
- Existing brand assets and category conventions, if any exist.
- Epistemic ledger access — meaning claims about signs get tagged like any other claims.

## Expected outputs — the Semiotic System Spec

1. **Sign inventory with functional annotation.** Every core sign (name, marks, color, typography, sonic and verbal signatures, packaging gestures, spatial behaviors) annotated with the job it performs. A sign with no job is a candidate for deletion.
2. **Code strategy.** The category's semiotic conventions, each marked *adopt* (buys membership and legibility), *adapt* (membership with an accent), or *break* (buys difference, costs legibility). Adopting all codes is invisibility; breaking all codes is illegibility. The table shows the chosen balance and its rationale.
3. **References, myths, and provenance.** The cultural material the system draws on — with a rights check: is this material *ours to use*, by participation, origin, or license of the culture that made it? Borrowed depth reads as costume when the borrowing is illegitimate, and the audience that polices this is the audience premium brands need most.
4. **Narrative grammar.** The story patterns the brand tells and the archetypal positions it occupies — used as analytical instruments, not astrology. Specify: recurring protagonist, recurring tension, recurring resolution.
5. **Rituals and repeated gestures.** Actions the brand performs identically over time (openings, sign-offs, annual moments, service gestures) — meaning built through repetition rather than announcement.
6. **Distinctive-asset audit.** For each asset: fame (do people recognize it?) × ownability (do they attribute it to us alone?). Assets high on both are the estate; protect them from redesign enthusiasm. Assets low on both are dead weight.
7. **Meaning risks.** Polysemy and hostile readings: what else could each sign mean to an uncharitable, foreign, or adversarial reader? Semiotic drift: what happens to these signs at the hands of the organization's weakest executor?

## Quality bar

Every sign has a stated function tied to the meaning claim. The code strategy names what is deliberately conventional, and why. The system survives the **weakest-executor test**: a rushed regional team with a template produces recognizably on-system work.

## Definition of success

Creative and media teams can generate unlimited executions that accumulate into the *same* memory structure. Module 7 inherits an encoding system for its narrative architecture; Module 9 gets measurable assets (recognition, attribution) to track.

## Failure modes

- **Moodboard-ism.** Aesthetic coherence without semantic function — beautiful, jobless signs.
- **Code compliance.** Adopting the full category dress code and vanishing into it.
- **Reference without rights.** Depth borrowed from cultures the brand has no claim on.
- **Over-encoding.** A system so intricate only its author can reproduce it.
- **Novelty churn.** Redesigning working assets because the team is bored. Boredom inside the building is not evidence of wear-out outside it; distinctiveness compounds only if held.

## Never do

- Never invent cultural meanings ungrounded in Module 4's field map.
- Never encode claims the ledger's forbidden-claims register prohibits.
- Never sacrifice a famous, owned asset to a trend without pricing the memory loss in the ledger.

## Handoff

Complete when the Spec covers inventory, code strategy, provenance, grammar, rituals, asset audit, and risks — and its claims are tagged. Hands to Module 7 (encoding system for narrative and messaging) and Module 9 (asset metrics). If the field map shifts (4 reopened), re-check code strategy before 7 proceeds.

```
RISKS:    [sign] → hostile reading … | drift scenario …
```
